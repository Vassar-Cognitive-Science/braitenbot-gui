# Subsumption Architecture

So far we've built individual behaviors: drive toward light, avoid obstacles, reverse on bump. But a real robot needs all of these working together. What happens when the cruise behavior says "forward" and the avoidance behavior says "turn right" at the same time?

If you look back at the [obstacle avoidance tutorial](./obstacle-avoidance), you've actually already solved this problem once. The Constant node provides a forward drive, and the threshold + negative weight connections override it when an obstacle appears. That's two behaviors — cruise and avoid — with avoidance suppressing cruise through inhibition. You built a two-layer subsumption architecture without knowing it.

In this tutorial, we'll make that structure explicit, then extend it to three layers by adding the bump-and-reverse escape behavior from the [latch tutorial](./latches-with-delay). The key principle is simple: **higher-priority behaviors suppress lower-priority ones when they activate.**

**Time:** 30 minutes
**Prerequisites:** Completed [Obstacle Avoidance](./obstacle-avoidance) and [Latches with Delay](./latches-with-delay)

## Part 1: Two layers — Cruise + Avoid

Let's start by rebuilding the obstacle avoidance circuit and labeling the layers explicitly.

### The Cruise layer

The cruise layer is the robot's default behavior: drive forward at a constant speed. It's just a Constant node wired to both motors.

1. Add a **Constant** node, value **50**, label "Cruise"
2. Add two **Summation** nodes: "Left Mix" and "Right Mix"
3. Wire: Cruise → Left Mix (weight 1.0), Cruise → Right Mix (weight 1.0)
4. Wire: Left Mix → Left Motor (weight 1.0), Right Mix → Right Motor (weight 1.0)

The summation nodes are where behaviors combine. Right now only cruise feeds into them, so both motors run at 50.

### The Avoid layer

The avoid layer watches for obstacles and steers away. It uses the cross-wired threshold inhibition pattern from the earlier tutorial.

1. Add two **Analog Sensors**: "Left IR" (A0), "Right IR" (A1)
2. Add two **Threshold** nodes: "Left Detect" (threshold 60), "Right Detect" (threshold 60)
3. Wire each sensor → its threshold (weight 1.0)
4. Cross-wire with negative weights: Left Detect → **Right** Mix (weight **-1.0**), Right Detect → **Left** Mix (weight **-1.0**)

### How suppression works here

Walk through the signal values in trace mode:

**No obstacles** — both thresholds output 0. Each Mix node receives only the cruise signal: 50. Both motors at 50. The robot drives straight.

**Obstacle on the left** — Left IR reads 80, Left Detect fires at 100. The -1.0 weight sends -100 to Right Mix. Right Mix computes: 50 (cruise) + (-100) (avoid) = **-50**. The right wheel reverses while the left stays at 50. The robot pivots right, away from the wall.

This *is* suppression: the avoid layer's -100 overwhelms the cruise layer's +50, taking effective control of the right motor. Cruise is still running — it's still contributing +50 — but the avoidance signal is strong enough to override it. The cruise layer is suppressed, not turned off.

**Obstacle clears** — the sensor drops below threshold, the -100 disappears, and cruise's +50 takes over again seamlessly. No transition logic needed.

This is the essence of subsumption. Each layer runs independently and continuously. When a higher-priority layer has something to say, its signal is strong enough to dominate the summation. When it goes quiet, the lower layer's signal is all that's left.

## Part 2: Adding Escape — the third layer

The two-layer system handles most situations, but sometimes the IR sensors miss an obstacle — a thin chair leg, a dark surface the IR doesn't reflect from — and the robot drives right into it. That's where the bumper comes in.

The escape behavior is: **when the bumper is pressed, reverse both motors for 2 seconds, overriding everything else.** This is the highest-priority layer. It must suppress both cruise and avoid.

### Building the escape latch

We need the latch circuit from the [latch tutorial](./latches-with-delay): a bumper press is captured and held for 2 seconds.

1. Add a **Digital Sensor**: "Bumper" (pin 2, INPUT_PULLUP)
2. Add a **Summation** node: "Escape Latch"
3. Add a **Delay** node: set to **2000 ms**
4. Wire: Bumper → Escape Latch (weight 1.0)
5. Wire: Escape Latch → Delay (weight 1.0)
6. Wire: Delay → Escape Latch (weight 1.0) — on this connection, set a **clamping transfer curve**: points (-100, 0), (0, 0), (1, 100), (100, 100). This keeps the feedback at exactly 0 or 100, preventing runaway.
7. Add a **Threshold** node: "Escape Active" (threshold 10)
8. Wire: Escape Latch → Escape Active

Now when the bumper is pressed, Escape Active outputs 100 for approximately 2 seconds, then returns to 0.

### Wiring escape into the mix

Escape needs to reverse both motors. Wire Escape Active → Left Mix (weight **-1.5**) and Escape Active → Right Mix (weight **-1.5**).

Why -1.5 instead of -1.0? Because escape needs to overpower *both* cruise and avoidance. In the worst case, cruise contributes +50 and avoidance contributes nothing (no obstacle). To reverse the motor we need the total to be negative: 50 + (-150) = -100. With -1.5, the escape signal of 100 becomes -150 after weighting, which overwhelms the +50 from cruise and drives the motors fully in reverse.

If avoidance is also active during escape, one motor might see +50 (cruise) + (-100) (avoid) + (-150) (escape) = -200, which clamps to -100. That's fine — the motor reverses at full speed regardless.

## Testing the complete system

Activate trace mode and walk through each scenario. The signal arithmetic at the Mix nodes tells you exactly what the motors will do.

### Scenario 1: Open floor (cruise only)

All sensors at 0. No thresholds firing, latch inactive.

| Input to Left Mix | Value |
|---|---|
| Cruise | +50 |
| **Total** | **50** |

Both motors at 50. The robot drives forward.

### Scenario 2: Wall on the left (avoid suppresses cruise)

Left IR at 80 → Left Detect fires at 100.

| Input to Right Mix | Value |
|---|---|
| Cruise | +50 |
| Left Detect (cross-wired) | -100 |
| **Total** | **-50** |

Left motor stays at 50, right motor at -50. The robot pivots right. Avoidance has effectively taken over the right motor.

### Scenario 3: Bumper hit (escape suppresses everything)

Pulse the bumper. Escape Active fires at 100 for 2 seconds. No obstacles present.

| Input to Left Mix | Value |
|---|---|
| Cruise | +50 |
| Escape Active | -150 |
| **Total** | **-100** |

Both motors at -100. Full reverse for 2 seconds, then Escape Active turns off and motors return to 50.

### Scenario 4: Bumper hit *while* avoiding an obstacle

Bumper pressed and Left IR at 80 simultaneously.

| Input to Right Mix | Value |
|---|---|
| Cruise | +50 |
| Left Detect (avoid) | -100 |
| Escape Active | -150 |
| **Total** | **-200 → clamped to -100** |

| Input to Left Mix | Value |
|---|---|
| Cruise | +50 |
| Escape Active | -150 |
| **Total** | **-100** |

Both motors at -100. The escape behavior dominates. The avoidance layer is still active, but its contribution doesn't change the outcome — the robot is already reversing at maximum speed. Escape has effectively suppressed avoid.

### Scenario 5: Escape ends, obstacle still present

After 2 seconds, Escape Active drops to 0. Left IR still reads 80.

| Input to Right Mix | Value |
|---|---|
| Cruise | +50 |
| Left Detect (avoid) | -100 |
| Escape Active | 0 |
| **Total** | **-50** |

The robot immediately begins steering right — avoidance picks up seamlessly where it left off. There's no gap, no transition logic. The layers just produce their signals independently, and the summation sorts out which behavior dominates.

## Why this works

The key insight is that suppression doesn't require any explicit "if-then" logic or mode switching. Every layer runs all the time, producing its output signal into the shared summation. Priority is encoded in the *weights*:

- Cruise contributes +50 (moderate)
- Avoid contributes ±100 (strong enough to override cruise)
- Escape contributes ±150 (strong enough to override cruise + avoid combined)

The summation acts as the arbitration mechanism. Whichever layer has the strongest signal wins. When the strong signal disappears, the weaker one is automatically revealed.

This is subsumption's central virtue: **behaviors are independent modules that don't know about each other.** The avoid layer doesn't check whether escape is active. Escape doesn't turn off avoidance. They just produce signals, and the math sorts it out. You can add or remove layers without modifying the existing ones.

## The physical result

Put the robot in a room with furniture and watch:

1. It drives forward (cruise)
2. It approaches a table leg — IR sensors detect it and the robot veers away (avoid overrides cruise)
3. It clips a corner the sensors missed — bumper clicks — the robot reverses for 2 full seconds (escape overrides everything)
4. After reversing, it resumes driving forward. If the table leg is still nearby, avoidance kicks in again immediately

The robot navigates surprisingly well for something with no planning, no map, and no centralized control. Each behavior is a simple sensor-to-motor circuit. The competence comes from the layering.

## Extending the architecture

Once you have this pattern, adding new behaviors is straightforward:

- **Wander layer** (priority between cruise and avoid): add an Oscillator node with a symmetric dead-zone transfer curve feeding into the Mix nodes with weight ±0.3. The robot will gently weave left and right while cruising, covering more area. Avoidance overrides it because -100 > ±30.

- **Light seek layer** (priority between avoid and escape): add photoresistors with cross-wired connections to the Mix nodes, weight ±0.8. The robot drives toward light when no obstacles are in the way.

For each new layer, choose a weight magnitude that's stronger than the layers it should suppress and weaker than the layers that should suppress it.

## Packaging behaviors as compounds

The diagram has grown complex — 13 nodes. You can clean it up by grouping each behavior into a compound:

- **Escape** compound: contains the bumper, latch, delay, and threshold. Exposes one output: the -150 suppression signal.
- **Avoid** compound: contains the two IR sensors, two thresholds, and the cross-wired connections. Exposes two outputs: left and right inhibition signals.
